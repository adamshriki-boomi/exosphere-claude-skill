# Inclusive Language

**Words matter.** People are at the heart of everything we do, and we want to create an inclusive world within our Boomi community. **We are committed to using language that includes everyone.** We are making significant shifts away from terms that have been “common industry” terms, because we now recognize them as offensive, non-inclusive, or with historically racist implications.

## Gender inclusive language

-   [Guidelines](https://boomii.atlassian.net/wiki/spaces/BD/pages/15858204682/Use+Gender+Neutral+Language#Guidelines)
-   [Examples](https://boomii.atlassian.net/wiki/spaces/BD/pages/15858204682/Use+Gender+Neutral+Language#Examples)

## Guidelines

-   Don’t assume the CEO or the Developer or any others you may writer for is a “he”. Use third person and keep the language gender neutral. It’s OK to use *they, them,* or *their*.
-   We communicate to customers in the second person (you). We like to keep our communication direct.
-   If you are addressing someone and you know their pronoun, use that. If in doubt, use the person’s name.
-   If we have to ask our users to select a gender for themselves, provide the following three options (unless a specific state or federal regulation requires a binary choice):
    -   Female
    -   Male
    -   Non-binary/other

## Examples

**Nope:**Each Developer can create his own account in Atomsphere.

**Yes:**Developers can create their own account in Atomsphere.

**Nope:**When you add a new user his account details appear on the Admin dashboard.

**Yes:**When you add a new user, their account details appear on the Admin dashboard.

**Nope:**Every new user gets his/her own dashboard.

**Yes:**Every new user gets their own dashboard.

**Nope:**Hey guys!

**Yes:**Hey folks!
