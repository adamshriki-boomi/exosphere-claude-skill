# Empty States

#### Purpose

Textual element within a product when there is nothing to display to the user.

#### Function

Shows users that there is nothing to see or empty screen.

#### General use

-   For helping users decide what to do next and prevent user disappointment
-   For encouraging users to interact with the product
-   When there are no results in a search

#### Details

-   Whenever possible, refrain from negative language, and encourage the user to take action. Avoid words like can't and won't.
-   Add period if body of empty message is more than 2 lines, in most cases.
-   Use second POV directed at the user in encouraging, positive tone. Words like you, yours, etc.
-   Use the same phrasing as the button that populates the page. "No projects to see here yet. Create a new project."
-   Whenever possible, avoid exclamation points, long sentences and punctuation. Limit to a few lines of text.
-   Use links in empty states sparingly or when directing the user on how to "Create a new project" if not already indicated by a CTA
-   Always include and make sure CTAs push users to take action with a button
-   Start CTAs with an action word
-   Follow CTA and Microcopy guidelines for sentence case, tone, number of words, etc.

When nothing is available to show or take action on like in a cell, etc., use the following:

| Type | Usage |
| --- | --- |
| Not available | When date is uncertain or in the future |
| None | When data or field is left blank by user or system; not associated with dates |
