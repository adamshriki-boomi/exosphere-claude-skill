# Inclusive Language

**Words matter.** People are at the heart of everything we do, and we want to create an inclusive world within our Boomi community. **We are committed to using language that includes everyone.** We are making significant shifts away from terms that have been “common industry” terms, because we now recognize them as offensive, non-inclusive, or with historically racist implications.

## Terms we do not use

We will no longer use the following terms at Boomi due to their racist roots. This list will evolve over time and will be expanded to include other exclusionary language.

| Previous usage | Updated usage |
| --- | --- |
| Whitelabel | Custom labelling |
| Whitelist/Blacklist | Trusted/blocked (Allowlist/Denylist) |
| Grandfather | legacy, retired (depending on context) |
| Master Account (master/sub account pairing) | Primary account |
| Master process | Primary process |
| Master, excluding references to “master data" or "Master Data Hub" while those terms are under discussion | Primary |
| Whitehat | Ethical |
| Blackhat | Unethical |
| Segregate/Segregation | Separate/separation |
| Blackout | Restrict |
| Redline/redlining | Refuse/Refusing (or alternative, depending on context) |
| Seminal | Formative |
| Manhour/Man-hour | Person-hour |
| Manday/Man-day | Person-day |

## How you can help

-   Challenge and encourage one another to use more inclusive language.
-   Stop introducing new instances of problematic terminology into your work.
-   Identify other problematic language being used and let us know.
