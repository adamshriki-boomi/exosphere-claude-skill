# Field Labels

#### Purpose

Field labels are a design element for data entry found typically above a text field.

#### Function

Field labels are the names given to the text fields and capture user inputs in a form like Name, Address, etc.

#### General Use

For, When

-   For providing instructions and context about the type of information that should be entered
-   When a user needs to input information like date or name

#### Details

-   If there are more than 3 words in a title, use sentence case if applicable. If less than, use title case.
-   Field labels are brief and written in as few words as possible. Written in title case with no more than 2-3 words where possible. Use specific language like the name component or information requested for labels. Use placeholder and helper text sparingly as these pose cognitive and readability issues for users with accessibility challenges. Avoid articles like a, an, etc whenever possible.
-   Mark all fields as "Required" where mandatory and should ideally appear after error.
-   Mark fields as "optional" to lighten user load.
-   Written without punctuation.
