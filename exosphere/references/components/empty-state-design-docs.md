# Empty State

A way to help users navigate instances where there is no content or information

OverviewGuidelinesAccessibility

Empty states should be used in most cases where there is no data or content shown on the page. They help users decide what to do next and prevent the feeling of reaching a dead end.

Common use cases where empty states can be used include:

-   No matching search results
    
-   An empty area due to the user not having created anything yet
    
-   An empty table
